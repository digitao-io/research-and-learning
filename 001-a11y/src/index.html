2.2. Make Something Tab-able and Focusable

By adding tabindex="0", we can make everything tab-able and focusable. This paragraph is both tab-able and focusable.

By adding tabindex="-1", we will make an element not tab-able but focusable. To focus on an element with tabindex being set to -1, we have to use JavaScript.

In most of the cases, using the combination of specific tags or roles in HTML is already sufficient for challenge of making an accessible web page with keyboard. However, they cannot always cover everything in it. That's why sometimes we have to write JavaScript to implement the keyboard structure.

Sometimes, NVDA or other screen reading software will capture the key event. Since it is part of their functionality. This will cause our JavaScript not able to capture the keyboard event. To prevent this, use role="application" to tell NVDA, now we are in charge of everything. But if we do this, all the sub elements won't be read out by NVDA anymore. To prevent this, we can always add another role to the children elements, to make NVDA read it normally.

3. Screenreader Accessibility

3.1. Baseline

Generally speaking, if we use the correct HTML tag with semantic meaning, combined with correct and necessary roles, the web page will be also screenreader accessible.

3.2. Skip to Main Content

Most website has been built to put the menubar at the top of the page, it is actually logically correct. However, if a screenreader user is using this kind of page, he or she must go through all the content until he reaches the main content of the page. To overcome this problem, we can introduce a link at the beginning of the page, so that the user can directly jump to the content by pressing enter.

To implement this, simply add an ID to the container of the main content, and add an anchor link like href="#main-content" and that's it.
